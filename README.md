## 1. 小水力發電平台

本專案協助客戶進行小水力發電的監控與維運，整合「監控、分析、維運、收益管理」於同一系統，並延伸至多端應用（Web / Mobile PWA / Pad PWA / Line@）。

## 🚀 專案核心挑戰與解決方案 (Core Challenges & Solutions)

在開發這個系統時，最主要的技術挑戰在於 SCADA 未使用模組化開發，元件難以重用與維護：

*   **挑戰（Challenge）：** 各 SCADA 設備（馬達、閥門、電表等）邏輯耦合，缺乏共用機制，功能難以重用
*   **解決方案（Solution）：** 採用 Factory Pattern + 資料驅動
    *   將設備行為封裝為 class（物件導向）
    *   以設備參數清單初始化系統，自動生成對應設備實例
    *   抽離數據處理邏輯為 utils，提高重用性

## 💡 技術亮點與架構思考 (Technical Highlights)

*   **【SCADA Canvas 即時渲染】：** SmallHydroScada.vue + Konva.js 為整體系統中技術複雜度最高的模組，負責即時監控畫面的渲染與更新。
    *   採用三層 Canvas 架構（bottom / middle / top），將背景、設備與文字分層管理，提升渲染效率與維護性
    *   使用 async / await 控制圖片資源載入順序，確保畫面初始化過程無閃爍問題
    *   MQTT 資料進入後進行差異比對（diff），僅更新變動的 Konva 物件，優化效能
    *   在 onBeforeUnmount 中完整清除 timer 與 touch listener，確保 lifecycle 管理健全，避免記憶體累積
    *   此模組需同時整合 Canvas 繪圖、即時資料流處理與 Vue lifecycle，具備較高技術門檻。
*   **【Composable 封裝 MQTT Lifecycle】：** 
    *   透過 useMqttScada 封裝連線、訂閱、自動重連與斷線等流程，將 MQTT 連線流程模組化，提升重用性與可維護性。

## 2. 消防定位系統

這是專為極端環境設計的高即時性、高穩定度的戶外消防人員定位監控系統，整合水平/垂直定位、心率、氧氣瓶壓力及求救告警。

## 🚀 專案核心挑戰與解決方案 (Core Challenges & Solutions)

*   **挑戰（Challenge）：** 多人定位場景中，地圖資訊視覺重疊問題
*   **解決方案（Solution）：** 針對多人定位密集的場景，引入 D3 Force Simulation 計算動態防重疊機制 ，搭配視圖自動縮放（Fit All）與資訊卡切換機制，徹底解決視覺重疊痛點，大幅提升高壓環境下的資訊可讀性。

## 💡 技術亮點與架構思考 (Technical Highlights)

*   **【客製化地圖與座標系校正】：** 運用 Vue 3 結合 Leaflet 打造客製化 CRS（座標參考系）戶外平面圖，將真實世界公尺座標精準對應至地圖圖層，並實作圖片原點偏移校正與地圖旋轉，支援不同場景高度尺度伸縮需求。
*   **【高併發即時資料驅動】：** 整合 WebSocket 建立低延遲雙向通訊，即時接收人員定位、生理指標與告警事件，驅動地圖標記（Markers）與資訊卡片秒級動態更新。
*   **【多維度告警與語音調度系統】：** 實作多類型即時告警（SOS 求援、斷線、氧氣不足等），並設計語音廣播佇列機制（警鈴 + 編號自動排程播報），確保危急時刻資訊明確傳達。
*   **【高解耦即時地圖模組設計】：** 依據業務邏輯將複雜的地圖功能解耦，拆分為 8 個具備單一職責（Single Responsibility）的獨立 Composable 管理（如軌跡更新、圖層切換、告警狀態等）。成功精簡主元件的代碼體積，使各功能修改時互不干擾、限縮錯誤影響範圍，並顯著提升單一功能的除錯效率與後續的可維護性。

## 3. SCADA 編輯器

這是一個 No Code 拖曳式 SCADA 畫面編輯器，提供完整的「編輯 → 預覽 → 上線」流程，供工程師為各案場客製化工控監視畫面，無須手寫程式碼。


## 🚀 專案核心挑戰與解決方案 (Core Challenges & Solutions)

在開發這個系統時，最主要的技術挑戰在於巨量元件庫的效能與擴充瓶頸：

*   **挑戰（Challenge）：** 考量到元件庫未來數量龐大，若在編輯畫布初始化時一次性全量載入，容易造成主線程阻塞與畫面卡頓，且依賴硬編碼對照表管理路徑將導致架構臃腫、難以維護。
*   **解決方案（Solution）：** 參考開源架構設計，建立系統化歸檔規範，使選單階層直接對應實體資料夾路徑。透過動態按需載入（Lazy Loading）大幅降低初始載入資源、確保畫面流暢度，同時消除了硬編碼對照表，提升架構的維護便利與擴充彈性。

## 💡 技術亮點與架構思考 (Technical Highlights)

*   **【資料驅動與工廠模式 (Data-Driven & Factory Pattern)】：** 採用工廠模式定義標準化元件 Schema，將畫布上的圖形操作（如拖曳、縮放、巢狀群組等）完全解耦為結構化 JSON 資料。藉由 Vue 的響應式核心進行資料驅動渲染，在純 DOM 複雜環境中精準平衡了畫布的效能與架構的可擴充性。
*   **【發布訂閱模式 (Publish-Subscribe Pattern) 與生命週期管理】：** 針對 MQTT 即時大數據流，結合 Vue 路由守衛（Route Guards）與組件生命週期，實作頁面切換時自動訂閱與退訂（Subscribe / Unsubscribe）機制。建立 topicToWidget 高效索引表，確保數據更新時能精準、局部地觸發元件渲染規則，避免不必要的全局重繪（Re-render），大幅優化瀏覽器效能。
*   **【記憶體優化與資源生命週期控制】：** 在自訂素材功能中，嚴格管控 Object URL 與動態異步組件的生命週期，於專案關閉時主動釋放對象 URL（URL.revokeObjectURL）並清理 MQTT 訂閱線程。防止長期操作下的記憶體洩漏（Memory Leak）風險，確保工業級工控畫面長時間掛載的穩定性。
